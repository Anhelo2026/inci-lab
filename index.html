¿Cómo funciona M1?
1) Importas el lote → 2) cada producto entra a la Cola de validación con la marca ya normalizada (A.V.Y.N.A. → AVYNA) → 3) validas el INCI contra la foto, corriges lo que la IA leyó mal (cada corrección queda en la Bitácora con el build y modelo que la produjeron) → 4) pones tu sello y se guarda aplicando la regla de 3 casos: INCI idéntico se avisa, INCI cambiado guarda v2 con diff, producto nuevo guarda v1. Código y datos van separados: actualizar la app nunca borra tu catálogo.

Cola de validación

Productos importados esperando tu sello. Abre cada uno y coteja el INCI contra la foto.

Catálogo validado

INCIs con tu sello. Las recapturas de un mismo producto se guardan como versiones con diff (regla AVYNA).

Bitácora de correcciones

Cada diferencia entre lo que leyó la IA y lo que validaste tú, con el build y modelo responsables. Es el tablero de calidad del prompt del celular.

Respaldo y restauración

Mismo criterio que el celular: exporta todo a JSON; la restauración fusiona sin borrar.
